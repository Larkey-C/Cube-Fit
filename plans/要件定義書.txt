# 指示
あなたは熟練のフロントエンド・ゲームエンジニアです。
以下の仕様書に基づき、ブラウザで遊べるパズルゲーム「キューブフィット」を開発してください。
後からコードを拡張・修正しやすいように、クラスベース（オブジェクト指向）で設計し、HTML、CSS、JavaScriptのコードブロックを明確に分けて出力してください。

# ゲームの概要
- プロジェクト名：キューブフィット
- ジャンル：エンドレス・パズルゲーム
- コンセプト：1×1×2の直方体を転がし、指定された穴に起立させて（スッポリはめて）スコアを稼ぐパズル。

# 技術スタックとデザイン
- 描画：HTML5 Canvas を使用し、2Dアイソメトリック（等角投影法）で立体的に描画してください（上部頂点から斜めに見下ろした構図）。
- 依存関係：外部ライブラリ（Three.jsなど）は使用せず、Vanilla JSと標準Canvas APIのみで実装。
- デザイン：白、黒、グレーを基調としたミニマルでスタイリッシュなUI。

# 画面・UI仕様
1. **メイン画面（Canvas）**：3×3（1〜9）のグリッド盤面を描画。
   - 例：
     [1][2][3]
     [4][5][6]
     [7][8][9]
2. **UI情報**：現在のスコア、現在のモード（通常 / ブラインド）を画面上部に表示。
3. **メニュー画面**：ゲーム開始、スコアリセット、モード切替（通常 ↔ ブラインド）。

# ゲームルールと直方体の挙動（最重要ロジック）
1. **盤面と穴**：
   - 盤面は3×3の9マス。スタート時、1マスの「穴」がランダムに開く。
2. **直方体のサイズと状態**：
   - サイズは 1×1×2。
   - 状態は「立っている（1マス占有）」または「寝ている（2マス占有）」のいずれか。
3. **操作と転がり判定**：
   - [矢印キー]：直方体を上下左右に転がす。
   - 立っている状態から移動すると、その方向に倒れて寝た状態になる。
   - 寝ている状態から長辺方向に移動させると起立する。短辺方向に移動させると寝たまま平行移動する。
4. **はみ出しと無効操作（エッジケース処理）**：
   - **はみ出しの許容**：直方体が寝ている状態で、「1マス分が盤内にあり、もう1マス分が盤外にはみ出ている」状態は有効な配置として許可する。
   - **完全な盤外への移動禁止**：直方体の「全て」が盤面外に出るような操作はできない（落下ゲームオーバーの概念はない）。
   - **エラーアニメーション**：これ以上進めない方向（完全に盤外に出る方向）へ操作しようとした場合、直方体がその方向に「少しだけ傾いてから、元の位置にスナップして戻る」という視覚的なフィードバック（弾かれるアニメーション）を実装すること。
5. **クリア（穴入れ）とループ**：
   - 直方体が「起立した状態（1マス占有）」で、「穴」の座標と完全に一致した場合クリア。
   - スコアに+1し、シームレスに新しい穴と直方体をランダム再配置してゲームを続行する。
6. **その他の操作**：
   - [Space]：決定 / [Esc]：メニュー画面の開閉

# ブラインドモード仕様
視覚的な盤面（Canvas）を非表示にし、テキストだけでプレイするモード。
- 画面表示：Canvasを隠し、テキストログ（div要素など）で以下を表示。
  - 「現在の穴：[数字]」
  - 「直方体の位置：[数字]（立っている）、または [数字]と[外]（一部はみ出し）など」
- 矢印キーを押すたびにテキスト情報のみが更新され、プレイヤーの想像力で解かせる。

# 出力要件と品質チェック
- `Board`, `Cube`, `Game` など、役割ごとにクラスを分割して保守性を高めること。
- 転がり判定やアイソメトリック座標変換のロジックには、必ず日本語で詳細なコメントを残すこと。
- 省略なしで、そのまま実行できる完全なコードを出力すること。
